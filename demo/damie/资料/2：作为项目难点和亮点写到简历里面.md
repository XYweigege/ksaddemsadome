# 2：作为项目难点和亮点写到简历里面

主导前端工程化架构重构，基于 pnpm workspace 实现多包统一管理，构建自动化流程与模块化规范，提升研发效率与维护便利性。
采用 tsup、 Rollup 构建高性能基础库，业务模块基于 Vite 完成快速迭代与性能优化。
通过微前端架构整合主应用与子应用，实现独立部署与动态加载，支持多团队并行开发与版本解耦，全面提升系统扩展性与稳定性。



面试官可能会围绕这段话术展开深入的问题，一些高频问题和对应的最佳回答思路：

 🔍 1. 为什么选择 pnpm workspace 进行工程管理？相比 npm 和 yarn 有什么优势？ 

 🔍 2. 你们如何规范化自动化流程？具体有哪些自动化的场景？ 

 🔍 3. 为什么基础包选择了 tsup 和 Rollup，而业务包选用 Vite？ 

回答思路：

●tsup 基于 esbuild，构建速度极快，非常适合 TypeScript 基础库的打包，且支持 Tree Shaking。

●Rollup 更擅长处理库的打包，支持更多模块格式输出（ESM、UMD、CJS），对微前端子包的加载更友好。

●业务包使用 Vite，其基于原生 ESM 模块，支持热更新（HMR）与快速启动，开发体验优异，尤其在大型业务模块中能显著减少冷启动时间。

延伸补充： 可以分享一个构建时间对比的数据，比如使用 tsup 后构建时间缩短了多少秒，让面试官看到具体收益。

 🔍 4. 微前端是如何实现主应用和子应用的集成的？解决了哪些问题？ 

回答思路：

●使用 qiankun 或 Wujie 作为微前端解决方案，实现主应用和子应用独立部署、独立开发、动态挂载。

●主应用提供公共依赖和全局状态管理，子应用只关注自身业务逻辑，避免重复加载，减少包体积。

●子应用通过路由懒加载、动态注册到主应用，避免主应用首屏加载压力。

●支持多团队并行开发、独立发布、版本解耦，不影响整体稳定性。

延伸补充： 可以提到遇到子应用升级时的版本冲突问题是如何解决的，比如使用 Shadow DOM 或沙箱隔离。

 🔍 5. 在微前端架构下，主应用如何与子应用进行通信？ 

回答思路：

●采用 EventBus 或 CustomEvent 进行事件触发，主应用监听并响应子应用的消息。

●通过 PostMessage 实现跨窗口或 iframe 的消息同步。

●也可以基于 Shared Worker 共享线程，提供更高效的跨应用通信。

●同时配合 globalState 实现主子应用的状态同步。

延伸补充： 你可以补充一个具体场景，比如表单填写完成后，如何同步数据到主应用并展示。

 🔍 6. 如何保证多包管理下的版本一致性？ 

回答思路：

●使用 pnpm 的 workspace protocol 进行版本锁定，不同子包间共享依赖，避免版本冲突。

●配合 changeset 进行自动版本管理与变更记录，统一发布流程，减少人为干预。

●在 CI/CD 中加入版本检测与对比，自动处理依赖升级，防止发布失败。

延伸补充： 分享一个曾经解决过的版本冲突问题，以及如何通过 pnpm workspace 的 link: 机制快速修复。

 🔍 7. 如何优化微前端项目的首屏加载性能？ 

回答思路：

●主应用只加载必要的基础依赖，子应用异步加载，通过懒加载减少初始渲染时间。

●使用 Preload 和 Prefetch 提前加载静态资源，减少用户等待。

●基于 esbuild 的分包机制，将静态资源与业务模块进行独立打包，提升加载并发性。

●通过 Service Worker 和 Cache API 实现静态资源缓存，减少网络请求。

延伸补充： 可以提供一个具体优化前后的时间对比，例如首屏加载时间减少了 30%。

 🔍 8. 遇到子应用更新后导致主应用无法正常渲染，你是如何处理的？ 

回答思路：

●采用版本检测机制，主应用加载子应用前进行版本校验。

●通过 Submodule 的热更新策略，主应用不依赖子应用的固定版本，而是动态获取最新版本。

●如果检测到版本不兼容，通过 UI 提示用户刷新页面，或采用回滚机制加载上一版本。

延伸补充： 可以补充如何在开发阶段规避这些问题，比如使用 Mock 或自动化测试来提前发现。

 🔍 9. 在这种多包、多模块管理的工程下，如何保障代码质量？ 

回答思路：

●使用 ESLint、Prettier、Stylelint 进行静态代码检测，保证代码规范。

●引入 Jest 和 Vitest 进行单元测试，保障核心模块的正确性。

●在 CI 流程中引入 E2E 测试（Playwright/Cypress），避免版本更新带来的页面渲染问题。

●代码提交前强制执行 lint-staged 和单元测试，保证代码提交质量。

延伸补充： 可以分享一个具体的 Bug 是如何被自动化测试发现并拦截的。

 🔍 10. 未来你对这套工程化架构有什么优化计划？ 

回答思路：

●引入模块 Federation 优化微前端跨项目共享依赖，减少重复加载。

●增加自动化监控和告警机制，对构建失败、依赖冲突等问题实时反馈。

●优化动态模块加载策略，提升首屏渲染速度。

●进一步拆分公共模块，减少主应用负载，提升渲染性能。
